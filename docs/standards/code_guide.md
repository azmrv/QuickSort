# Code Style Guide — QuickSort Project

**Версия:** 1.0  
**Дата:** 2026-07-11  

## 1. Общие принципы
- Читаемость важнее краткости.
- Самодокументируемый код предпочтительнее комментариев (но комментарии обязательны для нетривиальной логики).
- Следовать идиомам языка.

## 2. Rust Style
**Именование:**
- Типы: `PascalCase`
- Функции/переменные: `snake_case`
- Константы: `SCREAMING_SNAKE_CASE`
- Модули: `snake_case`

**Структура файла:**
1. Импорты
2. Константы
3. Структуры и реализации
4. Трейты
5. Функции
6. Тесты

**Ошибки:**
- В Domain использовать `thiserror`.
- В Application возвращать конкретные типы, логировать через `tracing`.

## 3. TypeScript / React Style
**Именование:**
- Переменные/функции: `camelCase`
- Компоненты/типы: `PascalCase`

**Структура компонента:**
- Хуки → обработчики → рендер.
- Использовать `React.FC<Props>`.
- Не использовать inline-стили для сложных компонентов.

**Стилизация:**
- Ant Design с переопределением через `theme`.

## 4. Комментарии и документация
- Публичные API: `///` в Rust, JSDoc в TS.
- Нетривиальная логика: комментарии на английском (почему).
- TODO: `// TODO: TASK-XXX — description`.

## 5. Форматирование
- `cargo fmt`, `cargo clippy -- -D warnings`
- `npm run prettier`, `npm run lint`
