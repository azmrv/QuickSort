# Refactoring Rules — QuickSort Project

**Версия:** 1.0  
**Дата:** 2026-07-11  

Пополнять концепт по мере нахождения новых эффективных правил и требований
## Правила работы с кодом концепт
1. Проверять наличие git репозитория
2. Если есть git разработку проводить с использованием git
3. Иcпользовать и сверяться с gitignore
4. При отсутвии git настоятельно рекомендовать его использование
5. При отсутвиие gitignore формировать gitignore на основе данных о проекте и испоьщовать его в последующем

## 1. Общие принципы
1. Безопасность прежде всего.
2. Прозрачность — все изменения документировать.
3. Постепенность — маленькие шаги с тестированием.
4. Совместимость — внешние API не ломать без предупреждения.

## 2. Процесс внесения изменений
### 2.1. Перед изменением
- Создать задачу.
- Изучить ADR и стандарты.
- **Поискать примеры использования** новых библиотек/API в Google.com.
- Написать план.

### 2.2. Во время изменения
- **Не удалять файлы целиком**, закомментировать старые строки с `// OLD:`.
- Добавлять новые строки с комментариями на английском, объясняющими причину.
- Фиксировать TODO: `// TODO: TASK-XXX — description`.

### 2.3. После изменения
- Запустить тесты.
- Обновить документацию (имена файлов, структуру, API).
- Внести запись в `CHANGELOG.md`.

## 3. Правила для слоёв
- **Domain**: не добавлять внешних зависимостей, изменения обратимы.
- **Application**: использовать `tokio`, таймауты, избегать `unwrap()`.
- **Infrastructure**: атомарные операции с fs, кеширование.
- **Adapters**: тонкие обёртки над фасадом, `React.memo`.

## 4. Исключения
- Критические исправления — вне очереди, но с документированием.
- Эксперименты — в отдельных ветках.

## 5. Связь с документацией
- При TODO создавать задачу в `docs/workspace/tasks/`.
- При изменении структуры обновлять `SPECIFICATION.md` и `AGENTS.md`.

## 6. Чек-лист перед коммитом
- [ ] Код собирается.
- [ ] Тесты проходят.
- [ ] Линтеры чисты.
- [ ] Новый код прокомментирован.
- [ ] Старый код закомментирован.
- [ ] Документация обновлена.
- [ ] Задача для TODO создана.
