# Phase 3: Рефакторинг

**Дата начала:** 2026-07-11  
**Цель:** Исправить выявленные проблемы, улучшить код, не нарушая работоспособность.

---

## Правила выполнения

- **Не удалять файлы и содержимое файлов целиком.** Вместо этого комментировать старые строки, а новые добавлять с комментарием.
- **При добавлении или замене кода писать подробные комментарии на английском языке**, объясняющие «что» и «почему».
- **При работе с новыми библиотеками или API сначала искать примеры использования в Google.com** и только после понимания применять их.
- **Все TODO фиксировать в коде** в виде комментариев `// TODO: <описание>`, и одновременно заносить в документацию.

## Шаг 1. Рефакторинг Domain Layer

### 1.1. WindowsPath — проверка инвариантов
- **Файл:** `crates/quicksort-domain/src/value_objects/windows_path.rs`
- **Проблема:** Не проверяется наличие запрещённых символов в пути.
- **Исправление:**
  - Добавить проверку в конструктор `new()`.
  - Возвращать `Result<WindowsPath, ValidationError>`.

### 1.2. Folder — проверка имени
- **Файл:** `crates/quicksort-domain/src/models/folder.rs`
- **Проблема:** Имя папки может быть пустым.
- **Исправление:**
  - Добавить проверку в `new()`.

## Шаг 2. Рефакторинг Application Layer

### 2.1. ExecuteOperationUseCase — таймаут
- **Файл:** `crates/quicksort-application/src/use_cases/execute_operation.rs`
- **Проблема:** Отсутствует таймаут на выполнение операции.
- **Исправление:**
  - Обернуть в `tokio::time::timeout(Duration::from_secs(30), ...)`.

### 2.2. Facade — инициализация
- **Файл:** `crates/quicksort-application/src/facade.rs`
- **Проблема:** Не инициализируется при старте приложения.
- **Исправление:**
  - Добавить метод `init()` и вызывать его в `main.rs`.

## Шаг 3. Рефакторинг Infrastructure Layer

### 3.1. JsonConfigurationRepository — атомарность
- **Файл:** `crates/quicksort-infrastructure/src/repositories/json_configuration.rs`
- **Проблема:** Запись не атомарна (может произойти потеря данных).
- **Исправление:**
  - Писать во временный файл, затем атомарно переименовывать.

### 3.2. StdFileSystem — retry
- **Файл:** `crates/quicksort-infrastructure/src/file_system/std_fs.rs`
- **Проблема:** Нет retry-логики для временных ошибок.
- **Исправление:**
  - Добавить retry с экспоненциальной задержкой.

## Порядок выполнения

Рефакторинг идёт **снизу вверх** по слоям Clean Architecture:

1. Domain → 2. Application → 3. Infrastructure → 4. Adapters (Tauri + React) → 5. IPC/Shell Extension

Каждый шаг включает:
- Выбор задачи из `audit-issues.md` по приоритету (Critical → High → Medium → Low).
- Внесение изменений согласно правилам рефакторинга.
- Запуск тестов (`cargo test`, `npm test`) после каждого изменения.
- Обновление статуса задачи в `audit-issues.md`.
